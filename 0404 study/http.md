# HTTP는 무엇일까요?

## 요약

- **HTTP** : **H**yper **T**ext **T**ransfer **P**rotocol의 약자로 HTML 문서와 같은 리소스들을 가져올 수 있도록 해주는 프로토콜이다.

- HTTP의 동작방식은 클라이언트(웹 브라우저, 모바일)가 브라우저를 통해 어떠한 서비스를 URL을 통해 서버에 **요청(Request)**하면 서버에서는 해당 요청에 대한 결과를 **응답(Response)**하는 형태이다.

- HTTP의 메서드엔 GET, POST, PUT, PATCH, DELETE, HEAD, OPTIONS가 있다.

- 요청을 보낼 때 요청에 대한 정보를 담아 서버에 보내고, 서버도 응답할 때 응답에 대한 정보를 담아 클라이언트에 보내는데, 이를 **HTTP메세지**라고 한다.<br>
  HTTP 메세지는 시작줄, 헤더, 본문으로 구성된다.

- 요청은 시작줄, 헤더, 본문으로 구성되어 있으며 **시작줄은 메서드/주소/HTTP 버전** 순으로 이루어져 있다.<br>
  헤더엔 요청에 대한 정보를 담고 있으며, 본문에 요청할 때 함께 보낼 데이터를 담고 있다.

- 응답은 요청과 마찬가지로 시작줄, 헤더, 본문으로 구성되어 있다.
  첫 줄은 **버전/상태코드/상태 메세지**로 구성되어 있으며,
  두번째 줄부터 헤더로, 응답에 대한 정보를 담고 있다.<br>
  그 다음 본문이 위치해 있으며, 요청한 데이터를 담아 보내주기 때문에 응답엔 보통 본문이 있다. 응답 메세지에 담겨있는 리소스를 받아 브라우저가 화면에 렌더링한다.

<br>

## HTTP란?

**HTTP(Hyper Text Transfer Protocol)**는 HTML 문서와 같은 리소스들을 가져올 수 있도록 해주는 프로토콜이다.

- 프로토콜: 컴퓨터 내부에서, 또는 컴퓨터 사이에서 데이터의 교환 방식을 정의하는 규칙 체계

HTTP는 웹에서 이루어지는 모든 데이터 교환의 기초이며, **클라이언트-서버 프로토콜**이기도 하다.

- 클라이언트-서버 프로토콜: (보통 웹브라우저인) 수신자 측에 의해 요청이 초기화 되는 프로토콜

하나의 완전한 문서는 텍스트, 레이아웃 설명, 이미지, 비디오, 스크립트 등 불러온 하위 문서들로 재구성 됨

<img src="https://images.velog.io/images/m-vault/post/97142643-d0cd-4f61-ad35-31c19114b94f/1.jpg">

HTTP는 무상태 프로토콜이다.

이는 서버가 두 요청 간에 어떠한 상태나 데이터를 유지하지 않음을 의미한다.

(상태를 유지하기 위한 노력으로 **쿠키**와 **세션**을 사용한다.)

일반적으로 안정적인 **TCP/IP** 레이어를 기반으로 사용하는 응용 프로토콜이다.

동작방식은 클라이언트(웹 브라우저, 모바일)가 브라우저를 통해 어떠한 서비스를 URL을 통해 서버에 **요청(Request)**하면 서버에서는 해당 요청에 대한 결과를 **응답(Response)**하는 형태로 동작한다.

<br>

## HTTP 요청 메소드

HTTP 요청 메서드에는 다음과 같은 종류들이 있다.

- **GET**: 특정한 리소스를 받기 위한 요청. 리소스의 생성, 수정 및 삭제 등에 사용해선 안된다.
- **POST**: 리소스를 생성하거나 컨트롤러를 실행하는데 사용
- **PUT**: 변경 가능한 리소스를 업데이트 하는데 사용, 리소스 식별 정보를 포함해야 함
- **PATCH**: 변경 가능한 리소스의 부분 업데이트에 사용, 항상 리소스 식별 정보를 포함해야 함
- **DELETE**: 특정 리소스를 제거하는 데 사용
- **HEAD**: 클라이언트가 본문 없이 리소스에 대한 헤더만 검색하는 경우 사용
- **OPTIONS**: 클라이언트가 서버의 리소스에 대해 수행 가능한 동작을 알아보기 위해 사용

요청을 보낼 땐 요청에 대한 정보를 담아 서버에 보낸다.

서버도 응답할 때 응답에 대한 정보를 담아 클라이언트로 보낸다.

이러한 정보가 담긴 메세지를 HTTP 메세지라고 한다.

HTTP 메세지는 시작줄, 헤더, 본문으로 구성된다.

<br>

## 요청

실제 요청 HTTP 메세지를 살펴보면,

> GET https://www.zerocho.com HTTP/1.1 // 시작줄
> User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) ... // 헤더
> Upgrade-Insecure-Requests: 1<br>
> (본문 없음) // 본문

첫 줄이 시작줄이다.

- GET 메서드를 사용하고
- www.zerocho.com은 주소를 의미한다.
- HTTP/1.1 은 HTTP 버전이다.

즉, 시작줄은 **메소드 / 주소 / HTTP 버전**으로 이루어져 있다.

두번째 줄부터는 헤더이다.

요청에 대한 정보를 담고 있다.

헤더로부터 한줄 띄고부터 본문이 시작된다.

본문은 요청할 때 함께 보낼 데이터를 담는 부분으로, 지금은 단순 주소로만 요청을 보내고 있기 때문에 (본문 없음) 으로 본문이 비어있다.

<br>

## 응답

동일하게 응답을 살펴보면,

> HTTP/1.1 200 OK // 버전/상태코드/상태 메세지
> Connection: keep-alive // 헤더
> Content-Encoding: gzip
> Content-Length: 35653
> Content-Type: text/html; <br>

요청과 마찬가지로 시작줄, 헤더, 본문으로 구성되어 있다.

첫 줄은 **버전 / 상태코드 / 상태 메세지**로 구성되어 있다.

두번째 줄은 헤더로, 응답에 대한 정보를 담고 있다.

헤더로부터 한줄 띄고 난 다음엔 본문이 위치해있다.

응답에는 요청한 데이터를 담아 보내주기 때문에 보통 본문이 있다.

응답 메세지에 담겨있는 HTML을 받아 브라우저가 화면에 렌더링한다.
